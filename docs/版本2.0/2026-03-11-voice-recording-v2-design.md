# 第一部分：产品背景

## 一、版本定位

V1.0 完成了基础记账功能，用户可以手动记账、账户管理、预算控制。V2.0 的核心目标是**大幅降低记账门槛**，通过语音输入让记账像说话一样简单。

## 二、核心功能概览

| 功能 | 说明 |
|------|------|
| 语音记账 | 长按说话 → 自动识别 → 结构化记账 |
| 语音助手 | 简单问答式语音助手，支持查询财务数据 |
| 语音按钮 | 底部中心悬浮按钮，支持上滑选择意图 |

### 6.2 DeepSeek 解析 Prompt

```
你是财务记账助手。请从用户语音中提取记账信息。

用户语音："{voice_text}"

请以 JSON 格式返回以下信息：
{
  "type": "expense" | "income" | "transfer",
  "amount": 数字,
  "category": "分类名称",
  "account": "账户名称",
  "date": "YYYY-MM-DD 或 'today'",
  "note": "备注信息"
}

注意：
- type：如果提到"收入"、"赚钱"、"发工资"等为 income，"转账"为 transfer，其余为 expense
- amount：提取金额数字，如果没有提到金额，返回 null
- category：根据支出内容推断分类，如"吃饭"->"餐饮"，"打车"->"交通"
- account：根据提到的支付方式推断，如"微信"->微信钱包，"支付宝"->支付宝
- date：如果提到"昨天"、"上周"、"3号"等，转换为具体日期；默认 "today"
- note：提取有意义的备注信息

如果无法提取金额，返回：{"error": "无法识别金额"}
```

### 6.3 解析失败处理

| 场景 | 处理 |
|------|------|
| 无法识别金额 | 显示空白金额，让用户手动输入 |
| 无法识别分类 | 显示空白分类，让用户选择 |
| 无法识别账户 | 默认选择第一个账户 |
| 完全无法解析 | 提示"无法识别，请重试或手动记账" |

## 七、语音助手功能

### 7.1 功能定位

- V2.0 阶段：**简单问答**，仅回答用户问题，不执行操作
- 未来扩展：可执行操作的智能 Agent

### 7.2 支持的问答类型

| 类型 | 示例 | 返回 |
|------|------|------|
| 余额查询 | "我还有多少钱" | 总资产 / 指定账户余额 |
| 支出查询 | "我上个月花了多少" | 统计数字 |
| 分类查询 | "这个月餐饮花了多少" | 分类统计 |
| 预算查询 | "本月预算还剩多少" | 预算进度 |
| 目标查询 | "我的购车目标进度如何" | 目标进度 |
| 通用财务知识 | "什么是复利" | 知识回答 |

### 7.3 DeepSeek 问答 Prompt

```
你是财务助手。请根据用户的财务数据回答问题。

用户问题："{user_question}"

当前财务数据：
- 总资产：{total_assets} 元
- 本月收入：{monthly_income} 元
- 本月支出：{monthly_expense} 元
- 本月预算：{monthly_budget} 元
- 账户列表：{accounts}
- 目标列表：{goals}

请用简洁、自然的语言回答用户的问题。
如果用户问的是具体金额，请基于以上数据计算后回答。
如果数据不足，请说明"暂时无法查询该信息"。

回答应该：
1. 直接回应用户问题
2. 如有必要，给出简要分析
3. 保持友好、专业的语气
```

### 8.3 Executor 设计

```typescript
// 意图类型
type Intent = 'RECORD_EXPENSE' | 'RECORD_INCOME' | 'TRANSFER' | 'QUERY';

// 执行器接口
interface Executor {
  canHandle(intent: Intent): boolean;
  execute(params: Record<string, any>): Promise<ExecutorResult>;
}

// 记账执行器
class TransactionExecutor implements Executor {
  canHandle(intent: Intent): boolean {
    return ['RECORD_EXPENSE', 'RECORD_INCOME', 'TRANSFER'].includes(intent);
  }

  async execute(params: any): Promise<ExecutorResult> {
    // 创建流水记录
    const transaction = await createTransaction(params);
    return { success: true, message: '记账成功' };
  }
}

// 查询执行器
class QueryExecutor implements Executor {
  canHandle(intent: Intent): boolean {
    return intent === 'QUERY';
  }

  async execute(params: any): Promise<ExecutorResult> {
    // 查询数据并调用DeepSeek生成回答
    const answer = await generateAnswer(params.question);
    return { success: true, answer };
  }
}

// 执行器注册
const executors: Executor[] = [
  new TransactionExecutor(),
  new QueryExecutor(),
```

# 第五部分：数据设计

## 九、数据库表结构

### 9.1 语音记录表（voice_records）

```sql
CREATE TABLE voice_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  audio_text TEXT,           -- 识别的文本
  intent VARCHAR(50),         -- 识别的意图
  parse_result JSONB,         -- 解析结果
  is_executed BOOLEAN DEFAULT FALSE,  -- 是否已执行
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 9.2 API配置表（voice_settings）

```sql
CREATE TABLE voice_settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  provider VARCHAR(50),       -- 'baidu' | 'deepseek'
  config JSONB,                -- 加密存储的配置
  enabled BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

# 第七部分：非功能性需求

## 十一、性能要求

| 指标 | 要求 |
|------|------|
| 语音识别延迟 | < 3秒（网络正常情况下） |
| 解析延迟 | < 2秒 |
| 页面响应 | < 500ms |

## 十二、错误处理

| 场景 | 处理 |
|------|------|
| 网络异常 | 提示"网络不稳定，请检查网络" |
| 识别失败 | 提示"未能识别，请重试" |
| 解析失败 | 展示空白表单，允许手动填写 |
| API配额用完 | 提示"服务暂时不可用，请稍后再试" |

## 十三、安全考虑

- API Key 存储在后端，加密处理
- 音频数据不长期存储，识别后即删
- 用户数据隔离，每个用户只能访问自己的数据
