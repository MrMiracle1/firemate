## 二、交互设计

### 2.1 新增交易（记一笔）
- **入口**: 记账 Tab 底部"记一笔"按钮
- **形式**: 底部弹窗模态（slide up）
- **操作流程**: 选择类型 → 选择分类 → 输入金额 → 选择账户 → 选择时间 → 备注 → 确认
- **防重复**: 提交按钮点击后立即禁用，显示 loading 状态

### 2.2 查看交易详情
- **入口**: 首页"最近流水"、记账 Tab
- **形式**: 详情页面
- **显示内容**: 交易类型、金额、分类、账户、时间、备注

### 2.3 编辑交易
- **入口**: 交易详情页"编辑"按钮
- **形式**: 底部弹窗模态（复用记一笔组件）
- **支持修改**: 类型、分类、金额、账户、时间、备注

### 2.4 删除交易
- **入口**: 交易详情页
- **形式**: 删除按钮 + 确认弹窗
- **后端**: 删除时自动恢复账户余额

## 四、详细功能点

### 4.1 记一笔弹窗
- 弹窗高度: 90% 屏幕高度
- 背景: 半透明遮罩，点击关闭
- 动画: slide up 300ms
- 关闭方式: 点击遮罩/完成按钮/上滑

### 4.2 交易列表
- 分页加载: 每次 20 条
- 显示字段: 类型图标、分类、金额、账户、日期
- 点击: 跳转详情

### 4.3 交易详情页
- 顶部: 类型图标 + 金额（大字）
- 中部: 分类、账户、时间、备注
- 底部: 编辑按钮、删除按钮

### 4.4 防重复提交
- 提交按钮: `disabled={loading}` + 显示 spinner
- Store: 添加 `submitting` 状态

### 4.5 退出登录
- 位置: 更多页面底部
- 操作: 点击 → 确认弹窗 → 清除本地状态 → 跳转登录

## 五、数据流

```
用户点击"记一笔"
    ↓
底部弹窗出现 (add-modal.tsx)
    ↓
用户填写表单
    ↓
点击提交 → store.submitting = true → 按钮禁用
    ↓
调用后端 POST /api/transactions
    ↓
成功 → 关闭弹窗 → 刷新列表 → toast 提示
    ↓
失败 → toast 错误提示 → 按钮恢复
```
