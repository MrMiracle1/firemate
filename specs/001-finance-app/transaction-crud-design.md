## 0. 输入验证规则

| 字段 | 验证规则 | 错误提示 |
|------|----------|----------|
| 金额 | 必须 > 0，最多2位小数 | "金额必须大于0" |
| 日期 | 不能超过当前时间 | "日期不能超过今天" |
| 备注 | 最长128字符 | "备注不能超过128字符" |
| 账户 | 必选 | "请选择账户" |
| 分类（支出/收入） | 必选 | "请选择分类" |
| 转入账户（转账） | 必选，且不能与转出账户相同 | "请选择转入账户" |

## 1. 问题背景

当前问题：
1. `category_id` 是字符串（如 "food-1-1"），但数据库是 UUID 类型
2. 账户数据加载时机问题导致 `account_id` 可能是 undefined
3. 编辑时账户和分类混淆
4. 离线缓存和后端同步问题
5. 弹窗体验不佳

---

## 2. 核心设计原则

### 2.1 数据存储

- **分类**：存储在数据库（Supabase），用户可自定义
- **账户**：存储在数据库
- **交易**：存储在数据库

### 2.2 架构模式

采用 **Store + API 分离 + 混合模式 + 乐观更新**：

```
┌─────────────────────────────────────────────────┐
│                    Store                         │
│  • transactions[] - 交易列表（乐观更新）         │
│  • accounts[]     - 账户列表（乐观更新）         │
│  • categories[]   - 分类列表                     │
└─────────────────────────────────────────────────┘
                         ↑
            ┌─────────────┴─────────────┐
            │      API Service 层       │
            │  • transactionApi.create()│
            │  • transactionApi.update()│
            │  • accountApi.update()   │
            └──────────────────────────┘
```

## 5. 保存时的完整验证流程

```
用户编辑字段 → 关闭弹窗
        ↓
    验证输入
        ↓
   ┌──────┴──────┐
   ↓              ↓
验证通过      验证失败
   ↓              ↓
更新 Store   显示错误提示
+ 发起 API       + 阻止关闭
                  + 高亮错误字段
```

**验证时机：**
1. **输入时实时校验**（可选）
   - 金额：每输入一个字符
   - 备注：每输入一个字符
2. **关闭弹窗时校验**（必须）
   - 所有字段统一校验
   - 非法：不关闭弹窗，显示**错误提示展示：**
- 错误

弹窗底部红色文字提示
- 对应输入框边框变红
- 聚焦时显示具体错误

## 6. 数据流程

### 6.1 保存流程（乐观更新）

```
用户点击字段 → 打开编辑弹窗 → 用户输入/选择
        ↓
用户关闭弹窗 → 验证输入
        ↓
   ┌───────┴───────┐
   ↓               ↓
验证通过        验证失败
   ↓               ↓
更新 Store   显示错误提示
+ 调用 API       不保存
   ↓
   ┌───────┴───────┐
   ↓               ↓
API 成功        API 失败
   ↓               ↓
完成            弹窗提示"保存失败，是否重试"
```

### 6.2 代码示例

```typescript
// Store
const useTransactionStore = create<TransactionState>((set, get) => ({
  transactions: [],

  updateTransaction: async (id: string, updates: Partial<Transaction>) => {
    // 1. 乐观更新：立即修改本地 state
    set((state) => ({
      transactions: state.transactions.map(t =>
        t.id === id ? { ...t, ...updates } : t
      )
    }));

    // 2. 调用 API
    try {
      await transactionApi.update(id, updates);
    } catch (error) {
      // 3. 失败：回滚并提示
      await get().fetchTransactions(true);
      throw error; // 抛出错误让调用方处理
    }
  }
}));

// 详情页组件
const handleFieldUpdate = async (field: string, value: any) => {
  try {
    await updateTransaction(id, { [field]: value });
    // 成功：自动关闭弹窗
  } catch (error) {
    // 失败：弹窗提示
    Alert.alert('保存失败', '是否重试？', [
      { text: '取消', style: 'cancel' },
      { text: '重试', onPress: () => handleFieldUpdate(field, value) }
    ]);
  }
};
```

## 6. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 账户加载中点击编辑 | 弹窗提示"数据加载中" |
| API 请求超时 | 10秒超时，提示失败 |
| 网络离线 | 使用缓存数据，提示"离线模式" |
| 账户已删除 | 使用默认账户，显示提示 |
| 金额为0 | 提示"金额必须大于0" |
| 未选择必填项 | 高亮提示，阻止保存 |
| 快速连续点击 | 禁用按钮，Loading 状态 |

### 7.1 分类表

CREATE TABLE categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  type TEXT NOT NULL CHECK (type IN ('expense', 'income')),
  parent_id UUID REFERENCES categories(id),
  icon TEXT,
  is_default BOOLEAN DEFAULT FALSE,
